OpenClaw安装和飞书接入指南

一、前言

OpenClaw 是一个开源的 AI 助手网关，它能够将各种 AI 模型（如 GPT、Claude、Kimi 等）与你的日常工具连接起来。通过 OpenClaw，你可以让 AI 助手进驻飞书、钉钉、企业微信、Telegram 等平台，实现随时随地的智能对话。

本文将手把手教你：

  • 在 Ubuntu 服务器上安装 OpenClaw
  • 配置飞书机器人，让 AI 助手成为你的”数字同事”

阅读前准备：一台可以访问互联网的 Ubuntu 服务器（云服务器或本地虚拟机均可），以及一个飞书账号。

二、准备工作

2.1 硬件与环境要求

在开始之前，请确认你的环境满足以下条件：

项目 最低要求 推荐配置
操作系统 Ubuntu 20.04 LTS Ubuntu 22.04 LTS
CPU 1 核 2 核及以上
内存 512 MB 1 GB 及以上
硬盘 5 GB 可用空间 10 GB 及以上
网络 可访问公网 固定公网 IP 更佳

说明：OpenClaw 本身对资源消耗很低，但如果你计划同时运行多个 AI 模型或处理大量消息，建议适当提升配置。对于个人使用或小型团队，1 核 1GB 的配置已经足够流畅运行。

2.2 系统要求

OpenClaw 需要 Node.js >= 22 的运行环境。如果你的系统还没有安装，不用担心——OpenClaw 的安装器会自动帮你处理好这一切。安装脚本会检测当前 Node.js 版本，如果版本过低或不存在，会自动下载并安装合适的版本。

2.3 需要准备的账号

在开始部署之前，请确保你已准备好以下账号信息：

  1. 服务器登录账号：你需要拥有能够远程登录 Ubuntu 服务器的权限，通常通过 SSH 密钥或密码方式连接。如果你是使用云服务器（如阿里云、腾讯云、AWS 等），请确保已获取 root 或具有 sudo 权限的账号。

  2. 飞书账号：个人版或企业版均可用于创建机器人应用。个人版适合测试和学习用途；如果是企业环境使用，建议注册企业版以获得更完整的管理功能。

  3. AI 模型 API Key（可选）：OpenClaw 支持多种 AI 模型提供商：

    • 商业模型：GPT-4/GPT-3.5 (OpenAI)、Claude (Anthropic)、Kimi (Moonshot) 等，需要提前申请 API Key
    • 开源模型：Llama、Qwen 等可通过 Ollama 本地运行，无需 API Key

    如果暂时没有 API Key，也可以先完成部署，后续再配置模型。

安装过程通常需要 3-5 分钟，具体时间取决于你的网络速度和服务器性能。在此期间，安装器会自动完成以下工作：

4.2 创建飞书自建应用

要让 OpenClaw 能够接收和发送飞书消息，需要在飞书开放平台上创建一个企业自建应用。这个过程涉及多个步骤，请仔细按照以下指引操作。

步骤 1：登录飞书开放平台

打开浏览器，访问飞书开放平台：https://open.feishu.cn/app

使用你的飞书账号扫码或输入账号密码登录。如果你使用的是国际版 Lark（海外版飞书），请访问 https://open.larksuite.com/app，并在后续配置中设置 domain: "lark"。

步骤 2：创建应用

登录后，点击页面右上角的「创建企业自建应用」按钮。在弹出的表单中：

  1. 填写应用名称，建议使用描述性名称，例如：OpenClaw助手 或 AI智能助手
  2. 填写应用描述，简要说明用途
  3. 选择一个合适的应用图标（可以使用默认图标，也可以上传自定义图片）
  4. 点击「确定创建」完成应用创建

步骤 3：获取应用凭证

应用创建成功后，在左侧菜单中点击「凭证与基础信息」。在这个页面，你需要复制两个重要值：

  • App ID：格式类似 cli_xxxxxxxxxxxxxxxx，这是应用的唯一标识
  • App Secret：点击「查看」按钮后会显示一串随机字符，这是应用的访问密钥

⚠️ 安全提醒：App Secret 相当于应用的密码，请务必妥善保管，不要泄露给他人，也不要硬编码在公开代码中。

步骤 4：配置应用权限

为了让机器人能够正常收发消息并访问相关资源，需要在飞书后台配置相应的权限。在左侧菜单中点击「权限管理」，然后点击「批量导入」按钮。

在弹出的文本框中粘贴以下完整的权限配置（涵盖消息收发、文件操作、群聊管理等全部功能）：

点击「确定」保存，然后点击「批量开通」按钮申请这些权限。

权限说明：上述权限包括消息收发（im:message*）、群聊管理（im:chat*）、文件读写（aily:file*）、机器人菜单（application:bot.menu）等完整功能。如果只需要基础对话功能，可以仅保留 im: 开头的核心权限。

步骤 5：启用机器人能力

在左侧菜单中点击「应用能力」，然后选择「机器人」选项卡：

  1. 找到「机器人」开关并将其开启
  2. 在机器人名称字段输入显示名称，如 AI助手
  3. 可以选择配置机器人的头像和简介
  4. 点击「保存」按钮

步骤 6：配置事件订阅

为了让 OpenClaw 能够实时接收飞书消息，需要配置事件订阅。在左侧菜单中点击「事件与回调」：

  1. 在「接收方式」部分，选择「使用长连接接收事件」（WebSocket 模式）
  2. 在「事件订阅」部分，点击「添加事件」
  3. 搜索并选择 im.message.receive_v1（接收消息事件）
  4. 点击「保存」按钮

技术说明：长连接模式使用 WebSocket 协议保持与飞书服务器的持久连接，这种方式不需要公网 URL，也不需要配置反向代理，非常适合在普通云服务器上部署。

步骤 7：发布应用

最后一步是将应用发布到正式环境。在左侧菜单中点击「版本管理与发布」：

  1. 点击「创建版本」按钮
  2. 填写版本号，建议使用语义化版本号，如 1.0.0
  3. 填写更新说明，描述本次发布的功能
  4. 点击「保存」按钮
  5. 点击「申请发布」提交审批

对于企业自建应用，如果没有特殊限制，通常会自动通过审批。审批通过后，应用即可在飞书中使用。

六、常见问题排查

6.1 机器人在群组中不响应

如果机器人在群里不回复消息，请检查以下几点：

  1. 检查机器人是否在群中：确认机器人已被正确添加到目标群组
  2. 检查 @ 提及：默认情况下，机器人在群组中需要被 @ 才会响应
  3. 查看日志：运行 openclaw logs --follow 查看是否有错误信息
  4. 检查权限：确认飞书应用已开通 im:message.group_at_msg:readonly 权限

6.2 机器人收不到消息

如果私聊或群组消息都没有反应：

  1. 检查应用发布状态：在飞书开放平台确认应用已通过审批并发布
  2. 检查事件订阅模式：确认选择了「长连接」模式而非 HTTP 回调模式
  3. 检查权限完整性：对照文档确认所有必要权限都已开通
  4. 检查网关状态：运行 openclaw gateway status 确认网关正常运行
  5. 检查网络连接：确保服务器能够访问飞书的 WebSocket 服务器

6.3 App Secret 泄露

如果不小心泄露了 App Secret，应立即采取以下措施：

  1. 立即重置：在飞书开放平台的「凭证与基础信息」页面，点击 App Secret 旁边的「重置」按钮
  2. 更新配置：将新的 App Secret 更新到 OpenClaw 配置中
  3. 重启服务：执行 openclaw gateway restart 使新配置生效
  4. 审查日志：检查近期日志，确认没有异常调用

七、进阶配置

7.1 访问控制策略

OpenClaw 提供灵活的访问控制机制，可以根据实际需求调整。

私聊访问策略（dmPolicy）：

值 行为说明
"pairing" 默认模式。新用户首次联系机器人时会收到配对码，需管理员批准后才能对话
"allowlist" 仅允许白名单中的用户使用，其他用户会被拒绝
"open" 开放模式，允许任何人直接与机器人对话
"disabled" 禁用私聊功能，机器人只响应群组消息

群组访问策略（groupPolicy）：

值 行为说明
"open" 开放模式，任何群组成员都可以 @ 机器人（默认）
"allowlist" 仅白名单中的用户可以 @ 机器人
"disabled" 禁用群组消息响应

7.2 流式输出配置

飞书支持流式卡片输出，可以实时显示 AI 生成内容的进度，提升用户体验。

在配置文件中添加以下设置启用流式输出：

  • streaming: 启用文字流式输出
  • blockStreaming: 启用卡片块级流式更新

如果需要等待完整回复后再一次性发送（适合生成内容较短的场景），可以将这两项设置为 false。
