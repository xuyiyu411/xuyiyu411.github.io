OpenClaw 记忆机制深度解析：了解openclaw原生方案

引言

在 AI Agent 的工作流中，记忆（Memory） 是决定智能体能否持续学习、保持上下文连贯的核心能力。OpenClaw 提供了开箱即用的原生记忆方案，采用”文件即真相”的设计哲学，让智能体的记忆持久化变得简单而可靠。

设计理念

OpenClaw 的原生记忆系统遵循**”Files as Source of Truth”**的设计哲学：

“The files are the source of truth; the model only ‘remembers’ what gets written to disk.”

  • 纯 Markdown 存储：所有记忆以普通 Markdown 文件形式保存在工作区
  • 双层记忆结构：
    • memory/YYYY-MM-DD.md —— 每日日志（追加写入，只读今天+昨天）
    • MEMORY.md —— 精选长期记忆（仅主会话加载，群组环境永不加载）
  • 语义检索：通过向量嵌入实现语义搜索

技术架构

┌─────────────────────────────────────────────────────────────┐
│ OpenClaw 原生记忆层 │
├─────────────────────────────────────────────────────────────┤
│ 存储层 │
│ ├── MEMORY.md (长期记忆) │
│ └── memory/*.md (每日日志) │
├─────────────────────────────────────────────────────────────┤
│ 索引层 (SQLite + sqlite-vec) │
│ ├── 文本分块 (~400 tokens, 80 tokens 重叠) │
│ ├── 向量存储 (768-1536 维, 依模型而定) │
│ └── BM25 全文索引 (FTS5) │
├─────────────────────────────────────────────────────────────┤
│ 检索层 │
│ ├── Vector Search (语义相似度) │
│ ├── BM25 Keyword Search (关键词匹配) │
│ └── Hybrid Search (混合检索, 可配置权重) │
└─────────────────────────────────────────────────────────────┘

核心特性

特性 说明
混合检索 向量语义搜索 + BM25 关键词搜索，兼顾同义理解和精确匹配
多提供商支持 OpenAI、Gemini、Voyage、Mistral、Ollama、本地 GGUF 模型
自动选择逻辑 未配置时按顺序尝试：local → openai → gemini → voyage → mistral
MMR 重排序 Maximal Marginal Relevance，减少结果冗余
时间衰减 按文件日期对旧记忆降权，优先返回近期内容

嵌入缓存 SQLite 缓存 chunk embeddings，避免重复计算
批量索引 OpenAI/Gemini/Voyage 支持 Batch API，大型回填更快更便宜
sqlite-vec 加速 使用 SQLite 虚拟表存储向量，避免加载到 JS 内存

批量索引（Batch Indexing）

对于大型记忆库的回填，OpenClaw 支持使用 OpenAI/Gemini/Voyage 的 Batch API：

优势：

  • 更快：异步处理大量嵌入请求
  • 更便宜：OpenAI Batch API 提供折扣定价

sqlite-vec 向量加速

OpenClaw 使用 sqlite-vec 扩展将向量存储在 SQLite 虚拟表（vec0）中，避免将所有嵌入加载到 JS 内存：

注意：如果 sqlite-vec 扩展缺失或加载失败，OpenClaw 会自动回退到 JS 实现的余弦相似度计算。

会话记忆搜索（实验性）

OpenClaw 支持将会话历史（JSONL 格式）也纳入记忆搜索范围：

特点：

  • 会话更新被去抖动后异步索引
  • memory_search 不会阻塞索引操作
  • 结果仅包含片段，memory_get 仍仅限于记忆文件
  • 会话日志存储在 ~/.openclaw/agents/<agentId>/sessions/*.jsonl

增量阈值（触发后台同步）：

  • deltaBytes: 100000 (~100KB)
  • deltaMessages: 50 (JSONL 行数)

额外记忆路径

支持索引工作空间之外的 Markdown 文件或目录：

限制：

  • 仅索引 .md 文件
  • 符号链接被忽略
  • 目录会递归扫描

本地嵌入自动下载

当使用本地模式 (provider: "local") 时：

  • 默认模型：hf:ggml-org/embeddinggemma-300M-GGUF/embeddinggemma-300M-Q8_0.gguf (~0.6 GB)
  • 自动下载：如果 GGUF 文件缺失，自动从 HuggingFace 下载到缓存目录
  • 断点续传：下载中断后支持续传
  • 原生构建要求：需要运行 pnpm approve-builds 并重建 node-llama-cpp

最佳实践

何时写入记忆

根据官方最佳实践：

  • 决策、偏好、持久事实 → MEMORY.md
  • 日常笔记、运行上下文 → memory/YYYY-MM-DD.md
  • 当用户说 “remember this” 时，必须写入磁盘（不要保留在 RAM）
  • 如需持久化，明确要求 bot 写入记忆

配置建议

  1. 小型项目：使用默认配置，依赖自动选择逻辑
  2. 隐私敏感：启用 provider: "local"，完全离线运行
  3. 大型知识库：启用批量索引 batch.enabled: true
  4. 长历史会话：启用时间衰减 temporalDecay.enabled: true
  5. 多样化结果：启用 MMR mmr.enabled: true
