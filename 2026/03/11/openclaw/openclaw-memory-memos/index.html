OpenClaw 记忆机制深度解析：MemOS 接入指南

文档版本：基于 MemOS 官方文档整理
适用对象：OpenClaw 用户

1. MemOS 简介

1.1 什么是 MemOS

MemOS 是面向 LLM 和 AI Agent 的记忆操作系统（Memory Operating System），由 MemTensor 团队开发。它统一了记忆的存储、检索和管理，为 AI 系统提供长期记忆能力，实现上下文感知和个性化交互。

核心定位

MemOS 整合了三类记忆，构建分层架构体系：

记忆类型 说明
参数记忆 模型权重中编码的隐性知识
激活记忆 推理过程中的动态上下文
明文记忆 显式存储的外部记忆（MemOS 主要管理）

通过这种设计，AI 系统能够灵活调用、更新和整合记忆内容，实现精准推理、自适应学习，并在不同任务场景中持续演化。

核心特性

  • 不仅能记，更能快：生产级记忆服务，毫秒级响应
  • 跨模型和跨设备互操作性：统一的记忆交换协议，支持云端、边缘、设备间自由共享
  • 可预测性和异步调度：智能预测和多策略调度，在响应速度和搜索精度间达到最佳平衡
  • 多场景部署模式：支持公有云、私有云、本地部署与混合架构

1.2 MemOS 产品形态

MemOS 为 OpenClaw 用户提供两种接入方式：

方式 产品形态 数据存储 官网链接
云插件 SaaS 服务 MemOS 云端 https://memos.openmem.net/cn/
本地插件 本地部署 本机 SQLite https://memos-claw.openmem.net/

2. MemOS 作用

2.1 OpenClaw 为什么需要 MemOS

OpenClaw 是一个多智能体框架，但默认情况下 Agent 是”无状态”的——每次对话都是独立的，无法记住之前的交流内容。这带来了以下问题：

问题 说明
上下文断裂 每次对话从零开始，用户需要重复背景信息
无法持续学习 Agent 无法从过往交互中积累经验
技能无法沉淀 完成的任务无法提炼为可复用技能
多 Agent 协作困难 不同 Agent 之间无法共享知识和经验

2.2 MemOS 带来的价值

接入 MemOS 后，OpenClaw Agent 获得以下能力：

能力 价值
持久记忆 每次对话自动保存，长期保留历史上下文
智能召回 每轮对话前自动检索相关记忆，注入上下文
任务总结 碎片对话归纳为结构化任务（目标/步骤/结果）
技能进化 从任务中提炼可复用技能，持续升级优化
多 Agent 协同 记忆隔离 + 公共记忆 + 技能共享，实现协同进化
可视化 Viewer Web 面板管理记忆、任务、技能、分析、日志

2.3 典型应用场景

  1. 长期对话助手：记住用户长期偏好与历史交流，让对话更连贯
  2. 多场景任务编排：跨窗口、跨工具的任务记忆，保持一致理解
  3. 智能体协同：多个 Agent 共享或隔离记忆，提高协作效率

3. MemOS 服务架构

3.1 云插件架构

云插件采用客户端-服务端架构，OpenClaw 通过 REST API 与 MemOS 云端服务通信。

工作流程

核心组件

组件 职责
OpenClaw Plugin 拦截 OpenClaw 生命周期，调用 MemOS API
MemOS Cloud API 提供 /search/memory 和 /add/message 接口
云端存储 分布式数据库，支持高可用、可扩展

认证方式

  • Token 认证：Authorization: Token <API_KEY>
  • API Key 获取：https://memos-dashboard.openmem.net/cn/apikeys/

3.2 本地插件架构

本地插件采用插件化单体架构，所有组件运行在 OpenClaw 进程内，通过 SQLite 作为统一存储。

架构分层

数据流向

核心组件

组件 职责 运行方式
Capture 过滤 system/self-tool 消息，剥离元数据 同步
Ingest 语义分片 → LLM 摘要 → 向量化 → 智能去重 → 存储 异步队列
Task Processor 检测任务边界 → 生成结构化摘要 异步队列
Skill Evolver 规则过滤 → LLM 评估 → SKILL.md 生成/升级 → 安装 异步队列
Recall FTS5+Vector 混合检索 → RRF → MMR → 时间衰减 → LLM 过滤 同步
Viewer Web 管理界面，7 个管理页面 HTTP 服务

检索算法

算法 说明
RRF (Reciprocal Rank Fusion) 融合 FTS5 和向量检索结果
MMR (Maximal Marginal Relevance) 平衡相关性和多样性，λ=0.7
时间衰减 14 天半衰期，新记忆权重更高
LLM 过滤 最终相关性过滤，阈值 ≥0.45

4. MemOS 部署和使用

4.1 云插件

部署步骤

第一步：获取 API Key

  1. 访问 MemOS Dashboard
  2. 注册/登录账号
  3. 进入 API Keys 页面
  4. 创建 API Key，记录密钥（格式：mpg-...）

第二步：安装插件

Windows 用户注意：如果遇到 Error: spawn EINVAL，请使用手动安装方式：

  1. 从 NPM 下载最新 .tgz
  2. 解压到 C:\Users\YourName\.openclaw\extensions\memos-cloud-openclaw-plugin
  3. 配置 openclaw.json 指向该路径

第三步：配置

方式一：环境变量（推荐）

方式二：.env 文件

按优先级读取以下文件：

  • ~/.openclaw/.env
  • ~/.moltbot/.env
  • ~/.clawdbot/.env

方式三：openclaw.json

使用方式

自动记忆和召回

安装配置完成后，以下操作自动进行，无需手动干预：

  1. 每轮对话前：自动调用 /search/memory 召回相关记忆，注入到 system 上下文
  2. 每轮对话后：自动调用 /add/message 保存对话内容

管理记忆数据

访问 MemOS Dashboard 进行：

  • 查看和管理记忆数据
  • 查看使用统计
  • 管理 API Key
  • 查看知识库

多智能体模式

启用方式：设置 "multiAgentMode": true 或 MEMOS_MULTI_AGENT_MODE=true

  • 自动捕获 ctx.agentId 实现数据隔离
  • 不同 Agent 拥有独立的记忆空间

配置项说明：

5. 云插件 vs 本地插件对比

维度 云插件 本地插件
部署难度 极低，即装即用 中等，需配置模型
运维成本 无 需自行维护
数据隐私 数据上云 完全本地
功能完整度 基础记忆 记忆+任务+技能+Viewer
检索性能 云端毫秒级 依赖本地硬件
离线使用 需联网 完全离线
